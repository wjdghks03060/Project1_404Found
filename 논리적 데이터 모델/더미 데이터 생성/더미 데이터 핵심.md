
## 💡 이 코드의 핵심 (중요!)
참조 무결성 (Foreign Keys)
```
 이 코드가 그냥 4개를 따로 만든 게 아니야.

Product_Master에서 만든 **product_id_list**를 Campaign_Master가 가져다 썼어.

Influencer_Master와 Campaign_Master에서 만든 inf_id_list, **campaign_id_list**를 Campaign_Performance가 가져다 썼지.

이렇게 **'관계'**가 있는 데이터를 만들어야 네가 하려는 'Seeding 평가'와 '성과 분석'이 가능해져. 이게 '데이터 모델링'을 코드로 구현한 거야.
```


```
현실성 (Logic)

Campaign_Performance를 만들 때, impressions(노출) > clicks(클릭) > conversions(전환)이 되도록 수치를 연동해서 만들었어.

이렇게 해야 나중에 네가 ROAS (revenue / actual_cost)나 CPC (actual_cost / clicks) 같은 진짜 KPI를 계산할 때 말이 되는 데이터가 나와.
```
